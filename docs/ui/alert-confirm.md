# 공통 컴포넌트 규칙 - Alert / Confirm

## 개요

`$alert`, `$confirm`은 프로젝트 전역에서 사용하는 비동기 모달 함수이다.  
브라우저 내장 `alert()` / `confirm()`을 사용하지 않고, **div 기반 모달(popup)**로 구현한다.

---

## `$alert(message: string): Promise<void>`

메시지 확인 창. 사용자가 모달을 닫을 때까지 다음 코드가 실행되지 않는다.

### 동작 규칙

| 항목 | 설명 |
|------|------|
| UI 형태 | div layout 모달 (overlay + 중앙 팝업) |
| 버튼 | `확인` 1개 |
| 반환값 | `void` (Promise 해소만 보장) |
| 닫힘 조건 | 확인 버튼 클릭, Esc 키, 닫기(X) 버튼, 모달 영역 밖 클릭 |
| **핵심 규칙** | `await` 시 모달이 완전히 닫힐 때까지 다음 행이 **절대 실행되면 안 된다** |

### 사용 예시

```ts
const createUser = async () => {
  // 저장 완료 작업 코드

  await $alert('저장되었습니다.');
  // 사용자가 모달을 닫을 때까지 여기서 대기

  // 페이지 이동 코드 (모달이 닫히기 전에 수행되면 안 됨)
};
```

---

## `$confirm(message: string): Promise<boolean>`

메시지 확인/취소 창. 사용자가 모달을 닫을 때까지 다음 코드가 실행되지 않으며, **boolean**을 반환한다.

### 동작 규칙

| 항목 | 설명 |
|------|------|
| UI 형태 | div layout 모달 (overlay + 중앙 팝업) |
| 버튼 | `확인`, `취소` 2개 |
| 반환값 | `확인` 클릭 → `true`, 그 외 모두 → `false` |
| 닫힘 조건 | 확인/취소 버튼 클릭, Esc 키, 닫기(X) 버튼, 모달 영역 밖 클릭 |
| **핵심 규칙** | `await` 시 모달이 완전히 닫힐 때까지 다음 행이 **절대 실행되면 안 된다** |

### 반환값 정리

| 사용자 행위 | 반환값 |
|-------------|--------|
| 확인 버튼 클릭 | `true` |
| 취소 버튼 클릭 | `false` |
| Esc 키 | `false` |
| 닫기(X) 버튼 | `false` |
| 모달 영역 밖 클릭 | `false` |

### 사용 예시

```ts
const createUser = async () => {
  // 유효성 검사 코드

  const isOk = await $confirm('저장하시겠습니까?');
  // 사용자가 모달을 닫을 때까지 여기서 대기

  if (!isOk) {
    return false;
  }

  // 저장 처리 (모달이 닫히기 전에 수행되면 안 됨)
};
```

---

## 구현 시 주의사항

1. **Promise 기반** — 모달 열림 시 새 Promise를 생성하고, 닫힘 이벤트에서 resolve 한다.
2. **확인 버튼만 `true`** — `$confirm`에서 확인 버튼 외 모든 닫힘 동작은 반드시 `false`로 resolve 한다.
3. **브라우저 내장 함수 사용 금지** — `window.alert()`, `window.confirm()` 사용하지 않는다.
4. **다중 호출 대응** — 동시에 여러 모달이 열리는 경우 z-index 순서를 보장한다.
5. **키보드 접근성** — Esc 키로 닫기, 포커스 트랩(focus trap) 적용을 권장한다.
