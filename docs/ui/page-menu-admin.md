# 관리자 페이지 유형 정의

> 각 페이지의 UI 유형, 구성 요소, 동작 방식을 정의합니다.

---

## 페이지 유형 분류

| 유형 | 코드 | 설명 |
|------|------|------|
| Dashboard | `DASHBOARD` | 통계 숫자 카드 나열 |
| List | `LIST` | 테이블 목록 + 검색 + 페이징 → **[페이징 리스트 UI 규칙](paging-list-ui-admin.md) 필수 준수** |
| Master-Detail 내 리스트 | - | Master-Detail 유형의 좌/우 목록도 동일 규칙 적용 |
| Detail | `DETAIL` | 단건 상세 조회 + 수정/삭제 |
| Tree | `TREE` | 트리 구조 표시 + 모달 CRUD |
| Master-Detail | `MASTER_DETAIL` | 좌측 목록 + 우측 상세/하위 목록 |
| Mapping | `MAPPING` | 좌측 선택 목록 + 우측 체크박스 트리 |
| Form | `FORM` | 단독 입력 폼 (로그인 등) |

## 1. 로그인 (`/login`)

| 항목 | 값 |
|------|-----|
| **유형** | `FORM` |
| **인증** | 불필요 (Public) |
| **API** | `POST /api/auth/login` |

- 중앙 정렬 로그인 카드
- 입력 필드: ID, PW

| **메뉴** | 회원 관리 > 회원 목록 |

| 3 | 회원 ID | `id` | - | |
| 4 | 회원명 | `name` | - | |
| 5 | 관리자 여부 | `isAdmin` | - | |
| 6 | 등록일 | `createdAt` | 가능 | |

## 4. 회원 상세 (`/member/detail/:usersSeq`)

| 항목 | 값 |
|------|-----|
| **유형** | `DETAIL` |
| **메뉴** | 회원 관리 > 회원 목록 (Breadcrumb에서 표시) |
| **API** | `GET /api/users/{usersSeq}`, `POST /api/users/create`, `POST /api/users/update/{usersSeq}`, `POST /api/users/delete/{usersSeq}` |

### 구성

- **모드**: 조회 / 생성 / 수정 (URL 파라미터로 구분)
  - `/member/detail/new` → 생성 모드
  - `/member/detail/{usersSeq}` → 조회 모드 (수정 버튼 클릭 시 수정 모드)

### 프로필 이미지 (폼 최상단)

- 폼 영역 최상단에 프로필 이미지 레이아웃 배치
- **구성**: 좌측 라벨("프로필 이미지") + 이미지 미리보기 영역 + 우측 하단 "업로드" 버튼
- **미리보기**: 등록된 이미지가 없으면 "없음" 텍스트 표시, 있으면 썸네일 표시
- **모드별 동작**:
  - 조회 모드: 이미지만 표시, 업로드 버튼 숨김
  - 생성 모드: 프로필 이미지 영역 숨김 (업로드 불가)
  - 수정 모드: 업로드 버튼 클릭 → 파일 선택 → 즉시 서버 업로드(`POST /api/file/upload`) → 업로드 완료 후 결과 이미지 즉시 반영 (저장 버튼과 무관하게 독립 동작)

- **폼 필드**:

| # | 라벨 | 필드 | 생성 | 수정 | 조회 |
|---|------|------|------|------|------|
| 1 | 회원 ID | `id` | 입력 | 읽기전용 | 읽기전용 |
| 2 | 비밀번호 | `pw` | 입력 | 입력(선택) | 숨김 |
| 3 | 회원명 | `name` | 입력 | 입력 | 읽기전용 |
| 4 | 관리자 여부 | `isAdmin` | 선택(Y/N) | 선택(Y/N) | 읽기전용 |

- **버튼**:
  - 조회 모드: 수정, 삭제, 목록
  - 생성 모드: 저장, 취소
  - 수정 모드: 저장, 취소

## 5. 메뉴 관리 (`/system/menus`)

| 항목 | 값 |
|------|-----|
| **유형** | `TREE` |
| **메뉴** | 시스템 관리 > 메뉴 관리 |
| **API** | `GET /api/admin-menus/tree`, `POST /api/admin-menus/create`, `POST /api/admin-menus/update/{adminMenusSeq}`, `POST /api/admin-menus/delete/{adminMenusSeq}` |

### 구성

- **트리 영역**: 1depth → 2depth 트리 구조로 표시
  - 드래그앤드롭으로 순서 변경 가능
  - 각 노드에 편집/삭제 아이콘 버튼
- **추가 버튼**: 1depth 추가, 2depth 추가 (선택된 1depth 하위)
- **CRUD**: 모달(Dialog) 방식
  - 생성 모달: 메뉴명, URL, 아이콘, 활성여부 입력
  - 수정 모달: 기존 데이터 로드 후 편집
  - 삭제: 확인 Dialog 후 처리

### 트리 노드 표시 정보

| Depth | 표시 항목 |
|-------|----------|
| 1depth | 아이콘 + 메뉴명 + 활성여부 배지 |
| 2depth | 메뉴명 + URL + 활성여부 배지 |

### 모달 폼 필드

| # | 라벨 | 필드 | 1depth | 2depth |
|---|------|------|--------|--------|
| 1 | 상위 메뉴 | `parentSeq` | 숨김 | 선택(1depth 목록) |
| 2 | 메뉴명 | `menuName` | 입력 | 입력 |
| 3 | URL | `menuUrl` | 숨김 | 입력 |
| 4 | 아이콘 | `menuIcon` | 입력 | 숨김 |
| 5 | 활성 여부 | `isActive` | 선택(Y/N) | 선택(Y/N) |

## 8. 권한별 메뉴 관리 (`/system/role-menus`)

| 항목 | 값 |
|------|-----|
| **유형** | `MAPPING` |
| **메뉴** | 시스템 관리 > 권한 관리 (별도 메뉴 또는 권한 관리 하위) |
| **API** | `GET /api/admin-roles/page`, `GET /api/admin-menus/tree`, `GET /api/admin-role-menus/page`, `POST /api/admin-role-menus/save` |

### 구성 (좌우 분할 레이아웃)

```
┌─────────────────┬──────────────────────────┐
│  좌측: 권한 목록  │  우측: 메뉴 트리 (체크박스) │
│                  │                           │
│  ● system       │  ☑ 대시보드                │
│    cms           │    ☑ 대시보드              │
│                  │  ☑ 회원 관리               │
│                  │    ☑ 회원 목록              │
│                  │  ☑ 시스템 관리             │
│                  │    ☑ 메뉴 관리             │
│                  │    ☑ 권한 관리             │
│                  │    ☑ 공통코드 관리          │
│                  │                           │
│                  │         [ 저장 ]           │
└─────────────────┴──────────────────────────┘
```

- **좌측**: 권한(역할) 목록 — 클릭으로 선택
- **우측**: 메뉴 트리 + 체크박스 — 선택된 권한의 메뉴 매핑 표시
- **저장 버튼**: `POST /api/admin-role-menus/save` 호출 (체크된 메뉴 일괄 저장)
- 좌측 권한 선택 변경 시 우측 체크 상태 갱신

## 9. 공통코드 관리 (`/system/codes`)

| 항목 | 값 |
|------|-----|
| **유형** | `MASTER_DETAIL` |
| **메뉴** | 시스템 관리 > 공통코드 관리 |
| **API** | `GET /api/common-codes/group/page`, `GET /api/common-codes/detail/page`, 그룹/세부 CRUD API 전체 |

### 구성 (좌우 분할 레이아웃)

```
┌──────────────────────┬──────────────────────┐
│  좌측: 그룹코드 목록    │  우측: 세부코드 목록    │
│                       │                       │
│  검색: [코드명] [검색]  │  (그룹코드 선택 시 표시) │
│                       │                       │
│  [+ 등록]              │  [+ 등록]              │
│  ┌──────────────┐     │  ┌──────────────┐     │
│  │ STATUS       │ ●   │  │ ACTIVE       │     │
│  │ CATEGORY     │     │  │ INACTIVE     │     │
│  │ ...          │     │  │ PENDING      │     │
│  └──────────────┘     │  └──────────────┘     │
│  페이징                │  페이징                │
└──────────────────────┴──────────────────────┘
```

- **좌측**: 그룹코드 목록 (검색 + 페이징 + 등록 버튼)
  - 행 클릭 시 우측에 해당 그룹의 세부코드 표시
  - 등록/수정/삭제: 모달
- **우측**: 세부코드 목록 (페이징 + 등록 버튼)
  - 그룹코드 미선택 시 "그룹코드를 선택하세요" 안내 표시
  - 등록/수정/삭제: 모달

### 좌측 테이블 (그룹코드)

| # | 컬럼명 | 필드 |
|---|--------|------|
| 1 | No | (순번) |
| 2 | 코드명 | `codeName` |
| 3 | 코드값 | `codeValue` |
| 4 | 순서 | `orderSeq` |

### 우측 테이블 (세부코드)

| # | 컬럼명 | 필드 |
|---|--------|------|
| 1 | No | (순번) |
| 2 | 코드값 | `codeValue` |
| 3 | 순서 | `orderSeq` |
| 4 | 등록일 | `createdAt` |
